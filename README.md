# TUSUR_Project
Квалификационная работа

Что Я хочу

Пользователь ставит задачи приложению:
Создай модель процесса в формате BPMN 2.0 на основании представленного документа
Ответ: файл в формате XML
Выдели сущности BPMN 2.0 из представленного документа
Ответ: таблица
Приложение формирует Промт.
Промт состоит:
Задача
Документ над которым выполняется задача
Контекст
Роль системы
Правила ответа
Контекст состоит:
Документа из базы документов релевантного Документу над которым выполняется задача 
Модели в формате XML соответствующей Документу из базы документов.
Нужно обеспечить для Пользователя:
выбор одной из двух задач
Загрузку в приложение своего Документа
Приложение должно:
Сформировать Промт
Выдать ответ пользователю.
Стэк
инференс LLM – ollama
модель LLM – Gemma3 (4b)
модель создания эмбеддингов – snowflake-arctic-embed2
векторная БД – Chroma
фреймворк для написания API – FastAPI
фреймворк для взаимодействыия с языковывми моделями и LLM – LangChain

Для реализации описанного функционала (архитектура RAG для генерации BPMN и извлечения сущностей) ниже представлена оптимальная структура проекта на FastAPI и LangChain.

## 🛠️ Назначение основных компонентов

## app/main.py
Конфигурирует FastAPI, подключает CORS-мидлвары и регистрирует роутеры из api/endpoints.py. Инициализирует подключение к векторной базе при старте.
## app/api/endpoints.py
Содержит два ключевых эндпоинта:

   1. POST /api/v1/process — принимает загруженный документ пользователя (UploadFile) и тип задачи (enum: CREATE_BPMN или EXTRACT_ENTITIES).
   2. POST /api/v1/ingest — административный эндпоинт для наполнения базы знаний (загрузка эталонного текстового документа вместе с его готовым XML-файлом).

## app/database/chroma_client.py
Инициализирует локальный клиент ChromaDB. Настраивает LangChainChroma с использованием модели эмбеддингов OllamaEmbeddings(model="snowflake-arctic-embed2"). При добавлении документов сохраняет текст в вектор, а метаданные связывает с путем к соответствующему XML-файлу.
## app/services/rag_service.py
Реализует логику поиска контекста:

   1. Принимает документ пользователя.
   2. Делает similarity_search через ChromaDB для поиска наиболее релевантного эталонного текста.
   3. Извлекает из метаданных найденного документа путь к его XML-модели и считывает её содержимое.
   4. Формирует структуру контекста.

## app/services/llm_service.py
Использует ChatOllama(model="gemma3:4b") от LangChain. Собирает финальный ChatPromptTemplate из пяти описанных вами компонентов (Задача, Документ, Контекст, Роль, Правила). Отправляет промпт в Ollama и возвращает чистый ответ (XML-строку или Markdown-таблицу).
## prompts/
Текстовые файлы, содержащие жесткие инструкции (Few-Shot) для модели Gemma3, чтобы она строго соблюдала синтаксис BPMN 2.0 XML или формат вывода таблицы, не добавляя лишних рассуждений в ответ.
